Client-Server

Séparer les préoccupations UI des préoccupations du stockage de données

Améliore la portabilité de l'UI sur toutes les plateformes

Améliore la scalabilité en simplifiant le composant serveur

Stateless

Chaque requête du client doit contenir toutes les informations nécessaires pour la comprendre

+ Visibilité: Monitoring

+ Fiabilité: Récupération sur echecs

+ Scalabilité: Pas de session

- Requête plus lourdes

- Session côté client

Cache

Une résponse doit indiquer si la resource est cacheable ou non

+ Supprime des interactions et améliore l'UX

- Fiabilité: Risque de données non-consitantes

Uniform Interface

Uniformiser les interfaces entre les composants

+ Architecture simplifiée

+ Meilleure visibilité des interactions

- Standardisation

Layered System

Permet d'avoir une architecture composée de couches hiérachiques

+ Un composant ne peut pas voir au delà de la couche avec laquelle il intéragit

+ Load balancing

- Latence
