REST Architecture

HTTP

Protocol

  • Client-Server
  • Stateless
  • Cacheable
  • Layered

Client-Server

Séparer les préoccupations UI des préoccupations du stockage de données
  • Portabilité des UI
  • Scalabilité

Stateless

Chaque requête du client doit contenir toutes les informations nécessaires pour la comprendre
  • Session côté client
  • Requêtes plus lourdes
  • Session côté client

Cacheable

Une résponse doit indiquer si la resource est cacheable ou non
  • Suppression d'interactions
  • Risque de données non-consitantes

Layered

Permet d'avoir une architecture composée de couches hiérachiques
  • Dissimulation de l'infra
  • Load balancing
  • Latence

Resources

  • URI
  • Uniform Interface

URI

Chaque resource doit être identifée avec un unique endpoint

Uniform Interface

Uniformiser les interfaces entre les composants en utilisant des datatypes standardisés
  • Ouverture
  • Standardisation

HATEOAS

Hypermedia as the Engine Application State
  • Séparation supplémentaire client/server
